- (event) Created product brief workspace and draft shell for auction_manager.
- (decision by user) Build a simple UI to track an ongoing auction for an amateur church cricket league.
- (context by user) The tool will run on one PC for the auction operator and be cast to a large display for league participants.
- (context by user) Last year the auction was managed in Excel; the new goal is an easier UI with simple buttons for auction management.
- (context by user) This is a fast-delivery internal tool; it will not be shipped or formally deployed.
- (option by user) Surface can be either a locally hosted web app or a desktop app.
- (decision by user) Auction format: randomized player list; each player reveal shows name, photo, role, and cost; team captains bid and highest bidder wins.
- (decision by user) Player cost starts as base price; during bidding the operator tracks the current bid.
- (decision by user) Track each team by name, logo, captain, remaining budget, total budget, and count by role.
- (decision by user) Operator controls include Start Auction, Next Player, Select Team, Increase Bid, Mark Sold, Mark Unsold, Undo, Close Auction, and Reset Auction.
- (context by user) Player/team data can be downloaded as CSV from Google Sheets; photos are local files sourced from Google Drive, with format chosen for the selected app surface.
- (decision by user) Auction rules: minimum bid increment 0.1, team budget cap 170, max squad size 13, role requirements Ace 2, Batting 3, Bowling 2, All Rounder 2, Girls 2, and unsold players return later.
- (event) Drafted first product brief from discovery inputs, keeping unresolved implementation choices as assumptions/open questions.
- (decision by user) First version surface is a locally hosted web app.
- (decision by user) Auction rules should be enforced as hard blocks, not warnings.
- (decision by user) CSV source columns are the Google Forms/Sheets registration export: Timestamp, Email address, Score, Place and Pastor Name, Full Name, Gender, Mobile Number, Email, Skill, TShirt Size, Jersey Number, Meal Preference, Photo Upload, Payment Confirmation, Payment Transaction Id, Validated.
- (decision by user) Player photo filenames contain the person name from the CSV, but exact naming is uncontrolled; supported image formats should include jpeg, png, heic, and webp.
- (decision by user) First version can mirror the operator view on the large display; a separate audience-facing screen can be developed later if needed.
- (decision by user) Unsold players should return automatically after all pending players are exhausted.
- (event) Reconciled user-provided open-question answers into the product brief and removed the resolved Open Questions section.
- (override) Minimum bid increment changed from 0.1 to 2 because the latest product decision replaces the earlier auction rule.
- (override) Unsold handling changed from automatic later auction pass to a second phase where the operator can randomly assign unsold players to teams after initial bidding.
- (event) Updated product brief to reflect revised bid increment and unsold-player second-phase random assignment decisions.
- (change) Updated product brief with default base price 13, local team list and logo folder inputs, file-based persistence, and no budget impact for random unsold assignments until stakeholder input changes these rules.
- (change) Clarified undo scope: multi-step undo covers live auction actions except Reset Auction and Close Auction; reset and close are dangerous operations that should not always be visible.
- (override) Stakeholder input replaces default base price 13 with role-based base prices: Batting 8, Ace 10, Bowling 6, Women 6.
- (override) Stakeholder input replaces random unsold-player assignment with operator manual assignment during the second phase.
- (change) Updated product brief to use role-based base prices and manual unsold-player assignment, with All Rounder pricing and manual-assignment budget impact left as deferred stakeholder inputs.
- (override) Stakeholder requirement replaces fixed auction-rule values with editable Auction Parameters entered or confirmed during Auction Creation; current base prices, bid increment, team budget, player cap, role targets, and manual-assignment budget behavior are defaults, not hard-coded rules.
- (override) Stakeholder requirement changes unsold handling: Phase 2 is randomized bidding for unsold players, and manual assignment moves to Phase 3.
