- (event) Started UX create run from final PRD at _bmad-output/planning-artifacts/prds/prd-auction_manager-2026-07-04/prd.md.
- (decision) Primary UX posture: calm under live-event pressure; the operator should always see the safe next action, captains should understand current auction state, and dangerous operations stay away from routine flow.
- (decision) Form factor: locally hosted responsive web app controlled from one PC and mirrored to a large display; mobile-first and separate audience display are out of scope for v1.
- (decision) Core IA phases: setup/import validation, live auction board and bidding, unsold assignment phase, state recovery/resume, and separated dangerous operations.
- (decision) Live board must show current player, photo or placeholder, role, base price, current bid, selected bidding team, team budget, remaining budget, squad size, and role counts with large-display legibility.
- (decision) Routine live controls include reveal next player, select bidding team, increase bid by +2, mark sold, mark unsold, undo, and random assignment; dangerous reset and close require separated access plus confirmation modals.
- (decision) Validation feedback should be blocking and specific during live sale or assignment: budget, squad capacity, and role target violations explain the reason quickly enough for the operator to recover during the event.
- (decision) Privacy floor: auction board displays only auction-relevant player data and never shows emails, mobile numbers, payment transaction IDs, or other registration fields.
- (assumption) Visual identity is not yet specified by the user; DESIGN.md should not finalize colors, typography, or component styling until Udeet provides direction or approves a fast-path assumption set.
- (decision) User selected Fast path: draft DESIGN.md and EXPERIENCE.md now with explicit [ASSUMPTION] tags and skip creative-tool mockups for this pass.
- (event) Fast-path draft completed for DESIGN.md and EXPERIENCE.md. Assumptions remain for visual identity, UI system, shortcuts, large-display layout feasibility, and some live-state edge behavior.
- (change) Updated UX spines against the final PRD and architecture: manual unsold assignment replaces random assignment; Current Bid uses configured role Base Price and configured Bid Increment; setup includes auction parameters and import severities; React/Fastify/Tailwind local event-mode and server-authoritative command behavior are now reflected.
- (change) Updated UX spines for PRD 2026-07-06 unsold-flow changes: Phase 1 unsold players now enter randomized Phase 2 rebidding with normal bidding controls; Phase 2 unsold players enter Phase 3 manual assignment; phase transitions, undo, resume, accessibility, and visual components now reflect the three-phase flow.
- (override) Supersedes earlier UX memlog decision that mentioned random assignment / direct unsold handling. Current PRD and UX require Phase 2 randomized player-order rebidding, followed by Phase 3 operator-selected manual assignment only.
- (change) Added Team rosters as a first-class UX surface: live Board/Rosters switching, all-Team current roster display, roster projection updates after sale/manual assignment/undo/resume, and Team rosters as the default final screen after Close Auction.
